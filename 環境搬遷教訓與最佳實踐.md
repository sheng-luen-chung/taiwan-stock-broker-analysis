# Python 專案跨電腦環境搬遷教訓與最佳實踐

## 1. 以前為什麼會遇到這些挫折？

- **虛擬環境（venv）路徑是絕對路徑**：VS Code/Python extension 會記錄 venv 的絕對路徑（如 H:\venvs\...），但每台電腦路徑都不同，導致找不到解譯器。
- **.venv 沒有跟著 repo 移動**：有時 venv 目錄沒被複製，或名稱不一致，導致環境失效。
- **.vscode/settings.json 設定了特定路徑**：只要路徑不同就會失效，VS Code 會報錯。
- **requirements.txt 編碼不一致**：Windows/macOS 預設編碼不同，導致 pip 安裝時出現亂碼或解碼錯誤。
- **沒有一鍵安裝腳本**：每次都要手動建立 venv、安裝套件，容易出錯。

---

## 2. 這次我們做了什麼？

- **移除 VS Code 設定中的絕對路徑**，讓 VS Code 自動偵測 .venv，不再依賴特定路徑。
- **統一虛擬環境名稱為 .venv 並排除在 .gitignore**，只要 clone 下來，執行腳本就會自動建立正確的虛擬環境。
- **提供跨平台一鍵安裝腳本（setup_venv.ps1 / setup_venv.sh）**，只要執行腳本就能自動建立 venv 並安裝 requirements.txt。
- **修正 requirements.txt 為 UTF-8 編碼**，保證 pip 在任何系統都能正確安裝套件。
- **安裝說明文件（INSTALL.md）明確指示步驟**，新手也能照著做，不會卡關。
- **移除 PowerShell 腳本中多餘的安全性設定**，避免出現不必要的警告訊息。

---

## 3. 以後怎麼做才不會再被 env 搞慘？

1. **永遠不要把 venv 目錄加入 git，只在本地建立。**
2. **每次換電腦或 clone 新 repo，只要執行一鍵安裝腳本（setup_venv.ps1 或 setup_venv.sh）。**
3. **requirements.txt 一律用 UTF-8 編碼。**
4. **VS Code 不要設定絕對路徑，讓它自動偵測 .venv。**
5. **遇到套件問題，先刪掉 .venv 重新執行安裝腳本。**
6. **安裝說明文件要清楚，讓任何人都能一鍵啟動。**

---

### 這樣做的好處

- **可攜性高**：任何電腦、任何路徑都能一鍵啟動。
- **維護簡單**：不用再手動設定 VS Code 或 Python 路徑。
- **新手友善**：只要照說明做，不會卡在環境問題。
- **團隊協作無痛**：每個人都能用同一套流程，減少「在我電腦可以跑」的問題。

---

**總結：**
以前的痛苦來自於「環境設定寫死」和「缺乏自動化」。這次我們徹底解決了這些問題，未來你只要 clone repo、執行安裝腳本，就能在任何地方順利執行，不再被 env 搞得很慘！
